### 13:09 飞书群聊消息发送错误纠正

**问题：** 连续多次使用错误的@格式在飞书群聊中发送消息失败

**错误做法：**
```
在message工具中使用 <at id="oc_6ef87b511b66b9d1d12296c6a3a89d99"></at> 格式
```

**用户反馈：**
> "仍然发送失败，你自己去搜索学习飞书群聊里如何向机器人和同事成功发送信息，不要让我再发现这个错误了"

**学习过程：**
1. 搜索飞书API官方文档
2. 阅读发送消息API文档（https://open.feishu.cn/document/server-docs/im-v1/message/create）
3. 阅读掘金教程（https://juejin.cn/post/7393185003205443622）

**关键发现：**
1. message工具不支持 `<at id="xxx"></at>` HTML格式
2. 飞书@用户需要使用富文本格式或卡片格式
3. @用户需要使用 open_id 或 user_id，不能用手机号或邮箱

**正确做法：**

使用富文本格式@用户：
```json
{
  "msg_type": "post",
  "content": {
    "post": {
      "zh_cn": {
        "title": "标题",
        "content": [
          [
            {"tag": "text", "text": "消息内容 "},
            {"tag": "at", "user_id": "ou_xxx"}
          ]
        ]
      }
    }
  }
}
```

**纠正措施：**
1. ✅ 创建知识文件：`memory/knowledge/feishu-group-messaging.md`
2. ✅ 更新MEMORY.md错误记录
3. ✅ 记录到每日日志
4. 🔄 未来发送飞书消息时使用正确格式

**错误严重性：** ⚠️⚠️⚠️ 高（用户明确要求不要再犯）

**反思：**
- 不能想当然地使用HTML格式
- 不同平台的API格式差异很大
- 遇到问题应该先查官方文档
- 错误纠正后必须记录到记忆系统
