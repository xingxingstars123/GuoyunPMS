# UI 分析与开发提示词

## 任务背景
我正在开发一个智能公寓/酒店管理系统（PMS），需要参考现有的 UI 设计。

## UI 参考图片说明
已上传的截图展示了一个典型的酒店管理系统界面，包含：

1. **顶部仪表盘**
   - 当日营业额显示
   - 可售房间数统计
   - 房态快速概览（预抵、已抵、预离、已离、新办）

2. **功能模块网格**
   - 日历房态
   - 单日房态
   - 订单管理
   - 客户列表
   - 餐饮管理
   - AI直播间
   - 智能定价
   - 智能评价
   - 更多功能

3. **渠道广告区**
   - 展示了 Booking.com 等 OTA 渠道的推广信息
   - 显示实时数据（新增房源数、已连接房源、单房交易额等）

4. **底部导航**
   - 首页、消息、房态、统计、我的

## 开发要求

### 技术栈
- **前端**: Vue 3 + TypeScript + Element Plus
- **后端**: NestJS + TypeScript + PostgreSQL
- **移动端**: uni-app（可选）

### 设计规范
- 使用现代化的蓝色渐变主题（参考图片配色）
- 响应式设计，支持桌面和移动端
- 图标使用清晰的象形图标（可使用 Element Plus Icon 或 IconFont）
- 数据可视化使用 ECharts

### 核心页面需求

#### 1. 首页仪表盘
```
请基于参考 UI 实现：
- 顶部数据统计卡片（营业额、可售房间、房态统计）
- 功能模块网格（8-12个核心功能入口）
- 快捷操作按钮
- 数据图表（可选）
```

#### 2. 房态日历页
```
需求：
- 月历视图，显示每间房的预订状态
- 颜色标识不同状态（可售、已订、入住中、清洁中）
- 点击日期可查看详情
- 支持快速锁房/解锁
- 支持拖拽调整订单
```

#### 3. 订单管理页
```
需求：
- 订单列表（支持筛选、搜索）
- 订单详情页
- 订单状态流转
- 支持修改、取消订单
- 打印功能
```

#### 4. 财务报表页
```
需求：
- 日/周/月切换
- 收入趋势图表
- 渠道收入对比
- 导出 Excel
```

## 使用图片的方式

### 方法 1: 在 Claude Code 中直接上传
```bash
# 启动 Claude Code
openclaw sessions spawn --runtime acp --agent-id claude-code

# 在会话中上传图片
# 然后发送消息："请分析这个 UI 并帮我实现类似的界面"
```

### 方法 2: 使用图片 URL
```typescript
// 如果图片已上传到服务器
const imageUrl = "https://your-server.com/ui-reference.jpg";

// 在 API 请求中引用
const response = await anthropic.messages.create({
  model: "claude-3-5-sonnet-20241022",
  max_tokens: 1024,
  messages: [{
    role: "user",
    content: [
      {
        type: "image",
        source: {
          type: "url",
          url: imageUrl
        }
      },
      {
        type: "text",
        text: "请基于这个 UI 设计图实现前端页面"
      }
    ]
  }]
});
```

### 方法 3: Base64 编码图片
```bash
# 将图片转为 base64
base64 /path/to/ui-screenshot.jpg > ui-base64.txt

# 在代码中使用
const imageData = fs.readFileSync('ui-base64.txt', 'utf-8');
```

## 调试与迭代

### 常见问题
1. **图片无法识别**
   - 确保图片格式为 jpg/png/webp
   - 图片大小不超过 5MB
   - 使用清晰的截图

2. **API 调用失败**
   - 检查 ANTHROPIC_API_KEY 是否正确
   - 确认网络可访问 api.anthropic.com

3. **代码生成不符合预期**
   - 提供更详细的需求描述
   - 分解任务，逐个功能实现
   - 使用明确的技术栈指定
